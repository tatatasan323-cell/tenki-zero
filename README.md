# tenki-zero ─ 転記ゼロの数字一元管理（小企業・自営業向け）

会社の帳票仕事の正体は、ほぼ「**同じ数字の転記**」です。
源流は〈**支払い・売上・勤怠**〉の3つだけ ―― そこで一度だけ記録し、あとの帳票は全部ここから出力します。
**数字の住所を1か所にすれば、転記という仕事は存在できません。**

## ▶ まず、ブラウザで体験（インストール不要）

**https://tatatasan323-cell.github.io/tenki-zero/**

サンプル内蔵。ボタン1つで「転記ゼロの月次締め」を体感できます（すべてブラウザ内だけ・送信なし・保存なし）。

そして本題はこちらです ―― この程度のものは、**あなたの会社のAIエージェントでも作れます**。
このリポジトリは“完成品の配布”ではなく、**自分たちで作るための見本（設計図・頼み方）** としてどうぞ。

- 完全ローカル（SQLite）。データはどこにも送信しません。ネットを切っても月次が締まります
- 判断は人（`masters/` の費目・仕分けルール）、適用は機械
- 自動化は**専門システム（会計ソフト・税理士）に渡す直前まで**。税務・給与の本演算はしません

## 使い方（月次の流れ）

**おすすめ: 受付画面から**

```
python app.py
```

ブラウザに “受付” が開きます。**経費・売上・勤怠の3つの窓口**に、届いた書類を
月末までに放り込んでおくだけ（何度でも追加OK・**同じ内容は自動でスキップ**、
同じ日の同じ数字も締めの時に自動排除）。
経費の窓口は **Excel(.xlsx)・PDFの請求書もそのまま** 受け付けます。
最後に月を選んで「**締める**」を押すと、帳票一式がその場でダウンロードできます。
手書きの書類は、スマホで撮って AI に「この表をCSVにして」→ 出てきたCSVを窓口へ。

**コマンド派はこちら**

```
1. inbox/ に放り込む（都度でも月末まとめてでもOK）
   ├ sales/       ← 売上集計ツール(#12)の「連携用CSV（日次×店舗）」
   ├ expenses/    ← 経費正規化ツールの「正規化CSV」／請求書CSV・Excel・PDFそのまま
   └ attendance/  ← 勤怠システムがエクスポートするCSV（SJISでも可）

2. 締める
   python close.py 2026-05

3. out/2026-05/ に帳票一式が出る
   01_月次損益.csv            … PL（売上高/原価/粗利/販管費/営業利益）
   02_経費内訳_費目別.csv
   03_売上計上_店舗別.csv
   04_勤怠集計.csv
   05_給与もと資料.csv        … 勤怠×単価（※所得税・社保の本演算はしない）
   06_部門別実績.csv          … 店舗別の売上・人件費率
   07_役員ダッシュボード.html … 数字＋グラフ＋自動分析コメント（開くだけ）
   08_税理士パック_仕訳候補.csv … 専門システムへの受け渡し＝ここが終点
   09_月次収支サマリ.csv

4. 人がやるのはこれだけ
   ・「未分類」が出たら masters/rules.csv に1行足す（翌月から自動）
   ・08_税理士パック を税理士さん／会計ソフトへ
```

## セットアップ ─ どうしても“この城ごと”欲しい方へ（3ステップ）

1. **Python を一度だけインストール** … https://www.python.org/downloads/ （インストール画面で「Add python.exe to PATH」に必ずチェック）
2. **このページ上部の緑の「Code」ボタン → 「Download ZIP」** でダウンロードし、ZIPを右クリック→「すべて展開」
3. 展開したフォルダの **`起動.bat` をダブルクリック** → ブラウザに受付画面（http://127.0.0.1:8760）が開きます

これは “あなたのPCの中だけ” で動く画面です（127.0.0.1＝自分のPCの住所）。
だからこそ、売上や給与の数字が **どこにも送信されません**。

コマンドに慣れている方は `python app.py`（受付画面）または `python close.py YYYY-MM` でも同じです。

**この城は“増築済み”でお届けします** ―― Excel（.xlsx）・PDFの取込と、Excel帳票（10_月次報告.xlsx）に必要な
“借り物”（外部ライブラリ）は `lib/` に同梱済み。**ダウンロードして起動すれば、そのまま全部動きます**（Windows 64bit）。

万一「Excel/PDFをスキップ」と表示される環境（他OS・32bit等）では、`増築.bat` をダブルクリック
（venv＝増築部屋を建てて借り物を入れ直します。コマンド派は `python -m venv .venv` → `.venv\Scripts\pip install -r requirements.txt`）。
その場合もCSV機能は常に動きます。
※ PDFで読めるのは「文字の入った表形式のPDF」です。スキャン画像・手書きは、AIに「この表をCSVにして」と頼んでCSVで投入してください。

サンプル（架空5店舗・2026年5月・1ヶ月分）を同梱しています。そのまま
`python close.py 2026-05` で動きを確かめられます。
`samples/請求書_受領分/` には “受け取ったままのバラバラ請求書”（SJIS・TSV・挨拶行・合計行・
**Excel・PDF**入り）もあるので、受付画面に放り込む所から体験できます。

## masters/（人が決める部分）

| ファイル | 中身 |
|---|---|
| items.csv | 費目（勘定科目相当）。自社の言葉で自由に |
| rules.csv | 取引先パターン → 費目 の対応表。**ここが“仕分けの判断”** |
| employees.csv | 従業員・所属・給与区分（月給/時給）・単価 |
| depts.csv | 部門（店舗） |

## やらないこと（線引き）

- 税務申告・年末調整・社会保険/所得税の本演算 → 専門家・専門システムの領域
- 会計ソフト内部の仕訳確定 → `08_税理士パック` を渡すまでが本システムの仕事
- 請求書の発行・受発注 → スコープ外（必要になったら育てる）

## 出力例・再生成

- 実行前に結果を見たい方へ: `samples/出力例_2026-05/` に帳票一式（役員ダッシュボード含む）を同梱しています
- `src/gen_samples.py` はサンプルデータの生成器（作者環境の元データに依存するため、読者はそのまま同梱の inbox/ をお使いください）
